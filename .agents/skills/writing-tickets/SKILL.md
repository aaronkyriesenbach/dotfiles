## What I do

- Use the load-work-info skill to make sure I have sufficient context
- Make sure that Jira tickets contain sufficient information for somebody with little context to execute
- Provide Acceptance Criteria at the bottom of the ticket detailing exact requirements to consider it complete
- Ensure that the ticket is assigned an appropriate epic
- Provide the entire planned ticket's contents for approval before creating or updating the ticket

## When to use me

Use this when you are creating or editing a Jira ticket.
Ask clarifying questions if more information is needed to create an acceptable ticket.
Tell the user if the Jira MCP server is not loaded, as it's required to retrieve context and create the ticket.
